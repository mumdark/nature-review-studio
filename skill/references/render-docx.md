# Docx render — payload schema and call site

> **V1.2.0 changes**: Rendering is style-driven through Word built-in headings and named body styles; fonts are Calibri with eastAsia = Microsoft YaHei, and ad-hoc run bolding is prohibited outside the approved bolding rules.

The docx is produced by `build/pipeline/render_review_docx.py`. The skill hands it a
JSON payload and a target path; the script handles fonts, headings, table rendering,
and bilingual heading selection.

## 1. CLI

```bash
python build/pipeline/render_review_docx.py ^
  --payload <tmp.json> ^
  --out <output.docx> ^
  --language zh | en
```

Word output defaults to uild/output/<case_id>_<YYYYMMDD>.docx; a mandatory
same-stem Markdown companion is written beside it. Failures exit 1
and write the failing byte location to stderr.

## 2. Payload schema (top-level)

```json
{
  "entry": "review | respond",
  "case_id": "string (no path separators)",
  "language": "zh | en",
  "editor_letter": {
    "revision_level": "Major revision | Accept with minor revisions | Reject in present form | Cannot be assessed",
    "paragraphs": ["string", "string", "..."]
  },
  "reviewers": [
    {
      "role_label": "Mechanism Reviewer | Statistical Reviewer | Clinical Validity Reviewer | Reproducibility Reviewer | Figures & Tables Reviewer | Writing & Clarity Reviewer",
      "overall_paragraph": "string (60–120 words in the selected language)",
      "overall_confidence": "high | medium | low",
      "major_concerns": [
        {
          "heading": "short noun phrase",
          "body": "string paragraph",
          "evidence_pointer": "section / figure / table",
          "confidence": "high | medium | low",
          "strategy": "add_textual_explanation | ..." ,
          "status": "DONE | DRAFTED | TODO_* | NOT_FEASIBLE | PROPOSED_DISAGREEMENT",
          "followup_risk": "low | medium | high"
        }
      ],
      "minor_concerns": [
        {
          "heading": "string",
          "body": "string paragraph"
        }
      ]
    }
  ],
  "consensus_block": [
    {
      "id": "Cc.N",
      "description": "short noun phrase",
      "raised_by": ["Reviewer A", "Reviewer B"],
      "axis": "clinical-validity | ...",
      "severity": "Major",
      "rationale": "1–3 sentence string"
    }
  ],
  "revision_task_table": {
    "headers": ["ID", "Reviewer", "Concern", "Strategy", "Status", "Input needed", "Output", "Blocks response?"],
    "rows": [
      ["T1", "Reviewer B", "...",
        "add_statistical_analysis",
        "TODO_ANALYSIS",
        "APOE call availability",
        "Re-weighted prevalence table",
        "Yes"]
    ]
  }
}
```

`respond` adds `strategy`, `status`, `followup_risk` and a per-reviewer
`revised_ms_pointer` field on each major concern.

## 3. Bilingual rendering

Render script selects the heading dictionary from `language`:

| English | Chinese |
|---|---|
| `Editor letter — <band>` | `编辑信 — <>` |
| `Reviewer X — <Role>` | `审稿人 X — <角色>` |
| `Overall` | `整体点评` |
| `Major concerns` | `主要问题` |
| `Minor concerns` | `次要问题` |
| `Cross-review consensus` | `跨审稿意见汇总` |
| `Revision task table` | `修订任务表` |
| `Confidence` | `置信度` |
| `Generated by nature-review-studio · <date> · <case id>` | `由 nature-review-studio 生成 · <日期> · <案例编号>` |

Technical terms stay verbatim regardless of language.

## 4. Page / style defaults

| Element | Style |
|---|---|
| Page | A4 portrait, margins 2.5 cm |
| Title | 16 pt bold |
| Section heading | 13 pt bold |
| `Overall` / `Major concerns` / `Minor concerns` heading | 11 pt bold |
| Numbered item heading | inline bold, 10.5 pt |
| Body paragraph | 10.5 pt, line spacing 1.15 |
| Table | `Table Grid`, header row 10 pt bold, body 10 pt |
| Footer | single-line, 9 pt italic |

If `language=zh`, the script switches the font to `Microsoft YaHei` for
Chinese runs (fallback to `Calibri` when not available).

## 5. Failure handling

Render script exits non-zero if:

- `payload.lint` rejects the schema.
- `editor_letter.revision_level` is not in the allowed band.
- Any reviewer is missing `role_label` or `overall_paragraph`.
- `revision_task_table.rows` is empty in `respond` entries.

Lint is performed by `render_review_docx.py:lint_payload`.
